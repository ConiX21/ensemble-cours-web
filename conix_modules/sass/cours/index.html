SASS Présentation

SASS est un langage de génération dynamique de feuilles de style initialement développé par Hampton Catlin et Nathalie Weizenbaum. Sass est un métalangage de feuilles de style en cascade (CSS). C'est un langage de script qui est interprété en CSS.

Installer gulp(local) dans NodeJS (npm install --save gulp)

1 - SASS Script

En plus de la syntaxe CSS simple, Sass prend en charge un petit ensemble d'extensions appelé SassScript. SassScript permet aux propriétés d'utiliser des variables, l'arithmétique et des fonctions supplémentaires. SassScript peut être utilisé dans n'importe quelle valeur de propriété
1.1 - Variables

La manière la plus simple d'utiliser SassScript est d'utiliser des variables. Les variables commencent par des signes de dollar et sont définies comme des propriétés CSS:
1.2 - Les types

SassScript prend en charge sept types de données principaux: numbers ( 1.2, 13, 10px) strings of text, with and without quotes ("foo", 'bar', baz) colors (blue, #04a3f9, rgba(255, 0, 0, 0.5)) booleans (true, false) nulls lists of values, separated by spaces or commas

1.3 - Opérations

Tous les types prennent en charge les opérations d'égalité (== et! =). En outre, chaque type a ses propres opérations qu'il a un soutien spécial pour.
1.4 - Interpolations

Vous pouvez également utiliser les variables SassScript dans les sélecteurs et les noms de propriétés en utilisant la syntaxe d'interpolation #{}
2 - Rules and Directives

Sass prend en charge tous les CSS3 @ -rules, ainsi que certains supplémentaires spécifiques à Sass connus sous le nom de «directives». Ceux-ci ont des effets différents dans Sass
2.1 - Import

Sass étend la règle CSS @import pour lui permettre d'importer des fichiers SCSS et Sass. Tous les fichiers SCSS et Sass importés seront fusionnés en un seul fichier de sortie CSS. De plus, toutes les variables ou mixins définis dans les fichiers importés peuvent être utilisés dans le fichier principal
2.2 - Media

Les directives @media dans Sass se comportent comme elles le font en simple CSS, avec une capacité supplémentaire: elles peuvent être imbriquées dans les règles CSS. Si une directive @media apparaît dans une règle CSS, elle sera émoussée jusqu'au niveau supérieur de la feuille de style, mettant tous les sélecteurs à l'intérieur de la règle. Cela facilite l'ajout de styles spécifiques aux médias sans avoir à répéter les sélecteurs ou à briser le flux de la feuille de style
2.3 - Extends

Il ya souvent des cas lors de la conception d'une page quand une classe doit avoir tous les styles d'une autre classe, ainsi que ses propres styles spécifiques. La manière la plus courante de manipuler ceci est d'employer à la fois la classe plus générale et la classe plus spécifique dans le HTML. Par exemple, supposons que nous ayons une conception pour une erreur normale et aussi pour une erreur grave.
2.4 - At-Root

La directive @ at-root provoque l'émission d'une ou plusieurs règles à la racine du document, plutôt que d'être imbriquées sous leurs sélecteurs parents
2.5 - Debug

La directive @debug imprime la valeur d'une expression SassScript dans le flux de sortie d'erreur standard. Il est utile pour déboguer les fichiers Sass qui ont compliqué SassScript en cours
2.6 - Warn

La directive @warn imprime la valeur d'une expression SassScript au flux de sortie d'erreur standard. Il est utile pour les bibliothèques qui doivent avertir les utilisateurs des dépréciations ou récupérer des erreurs mineures d'utilisation de mixin
2.7 - Error

La directive @error lance la valeur d'une expression SassScript comme une erreur fatale, y compris une trace de pile sympa. Il est utile pour valider les arguments des mixins et des fonctions
3 - Control Directives et Expressions

SassScript prend en charge les directives de contrôle et les expressions de base pour inclure des styles uniquement dans certaines conditions ou inclure le même style plusieurs fois avec des variantes.
3.1 - If

La fonction intégrée if () vous permet de dériver sur une condition et renvoie seulement l'un des deux résultats possibles. Il peut être utilisé dans n'importe quel contexte de script
3.2 - For

La directive @for renvoie plusieurs sorties d'un ensemble de styles. Pour chaque répétition, une variable de compteur est utilisée pour ajuster la sortie.
3.3 - Each

Il @each directive a généralement la forme @each $ var dans . $ Var peut être n'importe quel nom de variable, comme $ length ou $ name, et est une expression SassScript qui renvoie une liste ou une carte
3.4 - While

Il @while directive prend une expression SassScript et envoie à plusieurs reprises les styles imbriqués jusqu'à ce que l'instruction évalue à faux
4 - Mixin

Les Mixins vous permettent de définir des styles qui peuvent être réutilisés tout au long de la feuille de style sans avoir à recourir à des classes non sémantiques comme .float-left. Mixins peut également contenir des règles CSS complètes et tout ce qui est autorisé ailleurs dans un document Sass. Ils peuvent même prendre des arguments qui vous permet de produire une grande variété de styles avec très peu de mixins
4.1 - Defining a Mixin: @mixin

Les mixins sont définis avec la directive @mixin. Il est suivi du nom du mixin et éventuellement des arguments, et d'un bloc contenant le contenu du mixin
4.2 - Include

Les mixins sont inclus dans le document avec la directive @include
4.3 - Arguments

Mixins peut prendre les valeurs SassScript comme arguments, qui sont donnés lorsque le mixin est inclus et mis à disposition dans le mixin en tant que variables.
4.4 - Passer des blocs de contenu à un Mixin

Il est possible de passer un bloc de styles au mixin pour le placement dans les styles inclus dans le mixin. Les styles apparaîtront à l'emplacement de toute directive @content trouvée dans le mixin
